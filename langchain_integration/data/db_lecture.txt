Lecture — Introduction to Relational Databases

A relational database stores data in tables made up of rows (records) and columns (fields).
The most important concept is the primary key — a column whose value uniquely identifies each row.

A foreign key is a column that points to the primary key of another table. This is how relationships
between tables are represented. For example, an Orders table may have a customer_id foreign key
pointing to the Customers table.

The Entity-Relationship (ER) model is the planning step that comes before building tables. It models
the real world as entities (Student, Course) and relationships between them (Enrolls). The Enhanced
ER (EER) model extends this with inheritance, generalisation, and categorisation.

Normalisation is the process of organising tables to remove redundancy. The most common forms are
1NF (atomic values), 2NF (no partial dependencies), and 3NF (no transitive dependencies). A table
in 3NF has the property that every non-key column depends only on the primary key.

SQL is how you talk to a relational database. SELECT retrieves rows; INSERT adds them; UPDATE
changes them; DELETE removes them. To combine data from two tables you use a JOIN — most commonly
an INNER JOIN, which keeps only rows that match in both tables.
