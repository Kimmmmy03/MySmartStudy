 1. INTRODUCTION  
Higher education often requires first year students to adapt to a new learning culture 
that need independent study, structured understanding, and consistent self -
management. However, many first year IPG students still experience learning 
difficulties durin g this period, such as ineffective note taking, weak learning skills, 
and low motivation. These challenges can reduce study effectiveness and make it 
harder for students to manage learning materials, revise systematically, and maintain 
continue learning ha bits. 
In response to these issues, MySmartStudy is proposed as a digital learning platform 
designed to improve learning skills among first year trainee teachers at IPG Kampus 
Perempuan Melayu Melaka. The project aims to solve problems in existing study by 
digitizing learning and supporting cognitive development through structured learning 
tools, collaboration, and motivational elements such as gamification.  
In addition, the platform is designed to support lecturers through monitoring features 
and learning analytics so that student progress can be observed more effectively and 
feedback can be provided in real time. MySmartStudy is a platform that can be use  
through a website and mobile application, ensuring easiness and accessibility for 
both students and lecturers. Overall, the system is expected to provide a secure 
digital space for creating and organizing notes,

1.2 PROBLEM STATEMENT  
The development of MySmartStudy is by learning challenges faced by first -year IPG 
students during their transition into higher education. Many students struggle to 
adapt to a learning environment that requires stronger independence, better 
organization, an d consistent learning habits, which can reduce study effectiveness 
and academic performance. The key problems identified in this project include 
ineffective notetaking and retention, lack of monitoring and feedback, low motivation 
and engagement, limitatio ns of analogue tools, and disorganized self -study.  
1. Ineffective Notetaking and Retention  
Many first -year students face difficulty in producing structured notes that 
support understanding and long -term revision. When notes are incomplete or 
unorganized, students find it harder to revise and connect important concepts, 
which affects the learning  effectiveness.  
2. Lack of Monitoring and Feedback  
Students may not receive consistent monitoring and timely feedback during 
the learning process, especially when they are expected to learn 
independently. Without proper monitoring, students may only realize learning 
weaknesses when assessments occur, which  reduces opportunities for early 
improvement and academic guidance.  
3. Low Motivation and Engagement  
Low motivation and weak engagement can occur when students cannot see 
clear progress or feel that learning is repetitive and unsupported. When 
motivation decreases, students become less consistent in study routines, 
which directly impacts learning outcomes  and their ability to build strong 
learning habits.  
4. Limitations of Analog Tools  
Traditional paper -based learning methods and physical Thinking Maps can

 2.1 Overall Description  
MySmartStudy is a digital learning platform developed to enhance learning skills 
among first -year trainee teachers at IPG Kampus Perempuan Melayu Melaka. The 
system is proposed as a structured digital bridge to help students adapt to higher 
education learn ing culture by improving study organization, reducing over -reliance 
on rote memorization, and supporting more systematic learning habits.  
The platform is designed to solve problems in current learning practices by providing 
an organized environment where students can create, save, and manage study 
notes securely, while also supporting cognitive development through structured 
learning tools a nd collaboration features. MySmartStudy also integrates motivational 
support through gamification to encourage engagement and consistent learning 
habits, and it empowers lecturers through monitoring features and learning analytics 
so that student progress can be observed and supported more effectively.  
In terms of system delivery, MySmartStudy is scoped to be accessible through both 
a website platform and an Android mobile application, ensuring that students and 
lecturers can use the system conveniently across different devices. Overall, the 
system is ex pected to replace messy paper -based study methods with a more 
organized digital solution, improve the management of notes and Thinking Maps 
learning activities, and promote critical, reflective, and independent learning among 
users.

 2.1.1 Product Perspective  
From a real -world perspective, MySmartStudy addresses these issues by acting as a 
centralized platform where students can create, save, and organize study notes 
securely in one place, making learning easier to manage and access. It also 
supports learning s kill development by using structured learning tools such as 
Thinking Maps to improve cognitive organization, rather than requiring students to 
depend on separate external tools for visual mapping. This approach reduces 
hardness in learning because students  do not need to switch between multiple apps 
for notes, diagrams, and collaboration, which is a known weakness in existing 
applications.  
In terms of replacing existing practices, MySmartStudy does not merely replicate 
current tools but improves them by combining the strengths of existing applications 
into one system. Existing applications reviewed in the project show that some 
platforms sup port note organization but lack Thinking Maps, while others support 
visual mapping but lack progress tracking and lecturer monitoring. MySmartStudy is 
designed to address this gap by offering an integrated system that supports note 
organization, Thinking M aps, collaboration, progress tracking, and lecturer 
monitoring together, which provides a more complete learning ecosystem for the IPG 
context.  
Furthermore, MySmartStudy fits real -world academic needs because it also supports 
lecturers through monitoring features and learning analytics dashboards, enabling 
real-time feedback and progress visibility that are often missing when students learn 
indepe ndently without a tracking platform. The system is delivered as both a website 
and an Android application, ensuring accessibility and convenience for users 
settings. Overall, MySmartStudy is positioned as a practical digital solution that 
replaces ineffici ent manual learning organization with a structured, accessible, and 
supportive digital platform that strengthens learning habits and academic 
performance.

 2.1.2 System Interfaces  
MySmartStudy is designed as a multi -platform learning management system  that 
operates on both a web platform  and an Android mobile application , enabling 
students, lecturers, and administrators to access the system using commonly 
available devices. The updated system architecture consists of a Next.js web 
frontend , a Flutter mobile frontend , and a FastAPI backend API , with Firebase 
Authentication  used for login and Firebase Firestore  used for cloud data storage.  
 
In the latest version, the system also includes an AI engine powered by Google 
Gemini 2.5 Flash , which is integrated through the backend to support functions such 
as AI study companion chat, study material generation, timetable analysis, 
plagiarism detection, and grading assistance . This means the system now 
depends not only on Firebase services, but also on backend API communication and 
cloud -based AI processing.  
 
Since MySmartStudy uses cloud -based services for authentication, database 
storage, API communication, and AI features, users must have a stable internet 
connection  to log in, retrieve course content, submit work, receive notifications, and 
use AI -powered tools successfully. The web version is intended to run on any 
operating system that supports modern browsers, while the mobile version is 
developed using Flutter (Dart)  for Android devices. The web frontend communicates 
with the backend using Bearer token h eaders , while the mobile application 
communicates with the same backend through HTTP requests with Firebase ID 
tokens .

 2.4 Constraints  
1. Platform Constraint (Web + Android only)  
The system is constrained to operate as a web application  and an Android 
mobile application  only. The mobile platform is implemented using Flutter , 
but the current project scope and setup focus on Android usage, and support 
for other mobile platforms such as iOS is not included in the present 
implementation scope.  
 
2. Backend and Cloud Service Constraint (FastAPI + Firebase Dependency)  
The system is constrained by the availability and performance of its cloud -
connected backend architecture, which depends on a FastAPI backend , 
Firebase Authentication , and Firebase Firestore . User login, identity 
verification, data synchronization, and most core academic functions rely on 
these services being accessible and functioning correctly.  
 
3. AI Service Constraint (Gemini Dependency)  
In the updated version, the system is additionally constrained by the 
availability, performance, and usage limits of Google Gemini 2.5 Flash, since 
AI-powered features such as study companion chat, study material 
generation, timetable analysis, plagiarism detection, grading assistance, and 
educational image generation depend on this external AI service. If the AI 
service is unavailable, delayed, or limited by quota, AI -related functions may 
be affected.  
 
4. Internet Connectivity Constraint  
The system is constrained by the need for a stable internet connection , 
because both the web and mobile applications communicate with online 
backend, authentication, database, and AI services. Without internet access, 
users may be unable to log in, retrieve data, submit work, receive 
notifications, or access AI -assisted lear ning functions.  
 
5. Testing Constraint (Volunteer users and free tools)  
Testing and user evaluation are constrained by limited project resources, 
devices, and test users. Although the system is structured across web and 
mobile platforms, large -scale validation, stress testing, and long -term 
operational testing may be limited c ompared with a commercial production 
system.

consisting of a web application , an Android mobile application , and a shared

features are supported through Google Gemini 2.5 Flash . This design assumes that 
users access the system through supported devices with internet connectivity so that 
authentication, synchronization, course activities, messaging, and AI -assisted 
functions can operate correctly.

 3.1.3 Software Interface (MySmartStudy)  
i. Internet Connectivity  
● Internet access is necessary for MySmartStudy to function properly because 
the system relies on cloud -based authentication, a shared backend API, 
cloud data storage, and AI services . Core activities such as logging in, 
loading course content, saving mind maps, submitting assignments, 
synchronizing planner tasks, sending messages, receiving notifications, and 
using AI -powered functions require a stable connection to ensure data is 
stored and retrieved correctly.  
● Users must ensure they have steady internet access through Wi -Fi or mobile 
data to support smooth system performance, real -time updates, and 
consistent synchronization between the web and Android application, since 
both platforms communicate with the same backend and data layer. . 
 
ii. Web Application for Administration and Learning Management  
● MySmartStudy includes a web application  that supports academic learning 
management and platform administration. Through the web platform, lecturers 
and administrators can manage courses, announcements, resources, 
assignments, quizzes, question banks, gradebooks, attendance, groups, 
analytics, u ser accounts, audit logs, and homepage content.  
● The web platform also supports higher -complexity learning tasks such as 
mind map creation and editing  through an interactive editor, making the 
web environment the main interface for full feature access, detailed content 
management, and administrative control.  
 
iii. Mobile Application Interface (Android) for Flexible Access  
● MySmartStudy provides an Android mobile application  developed with 
Flutter (Dart)  to support flexible learning access for students and lecturers. 
The mobile interface allows users to access courses, assignments, quizzes, 
forums, messaging, notifications, planner tasks, attendance, grades, 
achievements, certificates, and AI -supported le arning tools through 
smartphones.

experience across devices.

both the web and mobile applications to the same business logic and data 
services. The backend verifies Firebase ID tokens , provides REST API 
endpoints, and coordinates data access across the system.  
● The system depends on Firebase Authentication  for secure login and 
identity verification, and Firebase Firestore  for cloud data storage and 
synchronization. This dependency ensures that learning materials, progress 
data, messaging, assessments, notifications, and other academic information 
remain consistent between platforms .

service interruption.

services.
